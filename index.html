Jira Dashboard Gadget Guide

Burn-up Forecasts

A quick guide to a burn-up chart: how much work is done, how much is still in scope, and when the team may finish.

Get Started View Screenshots

What It Does

See progress against the full amount of work

Burn-up chart

A burn-up chart compares completed work with the total planned work so you can see whether delivery is keeping up.

Forecast options

You can choose a simple best or average or worst-case view, or a Monte Carlo forecast with confidence ranges.

Flexible inputs

Scope and past delivery can come from manual values or from a saved Jira search using JQL.

Setup

Add the gadget to a Jira dashboard

  1. Open the Jira dashboard where you want to show progress and forecast information.
  2. Add the Burn-up Forecasts gadget to the dashboard.
  3. Open the gadget configuration screen.
  4. Set how the chart should behave and where the data should come from.
  5. Save and review the chart and finish-date guidance.

Configuration

Use two simple sections to set up the gadget

Setup

  • Text shown above the chart
  • Optional start date for projected finish dates
  • Forecast style: simple or Monte Carlo
  • Measure work by issue count or story points
  • Choose how many intervals to show and how long each one is

Forecast Data

  • Set scope manually or derive it from JQL
  • Use manual delivery history or JQL-derived history
  • Preview the last three completed intervals when JQL is used
  • Validation appears only when you try to save

Reading The Output

Read the chart in three parts

Scope line

The horizontal line shows the full amount of work to complete, either as issue count or story points.

Forecast lines

The forecast lines show likely progress based on past delivery, using either manual values or recent JQL history.

Completion guidance

The text under the chart explains when each forecast reaches the scope line and, if set, the projected finish date.

Key Caveats

Things to note

  • Very broad JQL searches can make the gadget slower.
  • If any of the last three completed intervals has no done work, JQL history cannot produce forecast lines.
  • Projected finish dates use the viewer's local timezone and a five-day working week.
  • Validation messages appear only after the first save attempt.
