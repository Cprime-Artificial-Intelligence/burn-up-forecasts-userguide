Jira Dashboard Gadget Guide

Burn-up Forecasts

Give your organisation a clear picture of total scope, track progress in real time, and forecast delivery dates with confidence — so teams can plan ahead and stakeholders always know where things stand.

Get Started View Screenshots

What It Does

See progress against the full amount of work

Burn-up chart

A burn-up chart compares completed work with the total planned work so you can see whether delivery is keeping up.

Forecast options

Choose the classic best, average, and worst-case view, or switch to Monte Carlo mode for Optimistic (P25), Likely (P50), and Conservative (P75) confidence lines based on 1,000 simulations.

Flexible inputs

Scope and past delivery can come from manual values, a typed JQL query, or a saved Jira filter. Measure work by work item count or story points.

Setup

Add the gadget to a Jira dashboard

  1. Open the Jira dashboard where you want to show progress and forecast information.
  2. Add the Burn-up Forecasts gadget to the dashboard.
  3. Open the gadget configuration screen.
  4. Set how the chart should behave and where the data should come from.
  5. Save and review the chart and finish-date guidance.

Configuration

Use five sections to set up the gadget

Setup

  • Text shown above the chart
  • Optional start date for projected finish dates and actuals overlays
  • Forecast Method: Classic (Best, Average, and Worst Case) or Monte Carlo
  • Measure work by work item count or story points
  • Choose how many intervals to show and how long each one is

Forecast Data

  • Set scope manually, type a JQL query, or pick a saved Jira filter
  • Use manual delivery history or automatically calculated history from the last three completed intervals
  • Preview matched work item counts and story points before saving when a query is used
  • Validation appears only when you try to save

Actuals Overlay

  • Plot Actual Scope — shows cumulative scope growth from work item created dates; requires a query-based total scope and a start date
  • Plot Actual Throughput — shows cumulative completed work from work item resolved dates; requires automatically calculated historical throughput and a start date
  • Both overlays use the configured start date to anchor where the lines begin on the chart

Scope Change Scenario

  • Optional Scope Change (%) field accepts a positive or negative whole number
  • The scenario always starts from your Total Scope value
  • Leave it blank or enter 0 to turn the scenario line off

Financial Configuration

  • Currency and Cost per Interval are both required to save
  • Cost per Interval must be a whole number greater than 0 (no decimals)
  • Supported currencies: USD ($), AUD (A$), NZD (NZ$), GBP (£), YEN (¥), YUAN (CN¥), EURO (€)

Reading The Output

Read the chart in three parts

Scope and scenario lines

The blue Total Scope line shows the full amount of work to complete, either as work item count or story points. When a scope-change scenario is configured, a lighter blue scenario line is added based on the configured percentage. When actual scope change is enabled, the Total Scope line grows over time from work item created dates.

Forecast lines

Classic mode shows Best Case, Average Case, and Worst Case lines. Monte Carlo mode shows Optimistic (P25), Likely (P50), and Conservative (P75) percentile lines. When actual throughput is enabled, a separate cumulative completed-work line is also shown.

Completion guidance

The guide beneath the chart names each forecast line's likely completion interval. When financial configuration is set, each forecast option also shows projected cost using your selected currency and interval cost. For beyond-range outcomes, the cost is shown as Beyond <symbol>Y. When a scenario is active, the guide also shows the signed scope change value. When a start date is set, each entry also shows a projected finish date in the format N week(s) Weekday dd/mm/yyyy. The guide header shows the start date and forecasting interval used in the calculation.

Key Caveats

Things to note

  • The gadget does not store historical data between views. Every time the gadget loads it fetches live data from Jira, so the chart always reflects the current state of your work items rather than a point-in-time snapshot.
  • Very broad queries can make the gadget slower because all matching work items are fetched and processed in the browser.
  • If any of the last three completed intervals has no done work, query-based history cannot produce forecast lines.
  • In story-point mode, work items without a Story Points value are counted in the work item total but flagged separately in the scope query preview.
  • Actual scope change and actual throughput overlays both require a query-based total scope (JQL or saved filter) and a configured start date.
  • Financial guidance appears only when both Currency and Cost per Interval are configured.
  • Cost per Interval must be a whole number greater than 0. Decimal values are rejected.
  • Projected finish dates use the viewer's local timezone and a five-day working week.
  • Validation messages appear only after the first save attempt.
